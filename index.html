Prework Study Guide

HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • !DOCTYPE declaration states the type of document the browser is expected to open
  • HTML attributes (lang="en") need to be defined at the beggining, not at the end of the element
  • Metadata tags include title, style, base, link, meta and script
  • Child elements are nested inside parent elements

CSS

  • Inline CSS - syle an element directly in the HTML file. Applies to specific HTML tag. Good for quick changes, but less flexible because each element needs editing individually.
  • Most frequent - external as it allows all design changes to be kept separate. It is brought in as a link element.
  • Separation of concerns principle - each section of code had its own responsibility
  • A CSS rule is formed of a selector and a declaration block. The selector defined the element or attributes to which the rules (declarations) will apply. Declarations are formed of a property and its value. Example Color is a property and blue is the value. Margin and padding are different properties. The margin states the amount of space around the outside of an element while padding refers to the space inside and element. The element gets bigger the more padding is added. CSS box model describes HTML elements as boxes with margins outside the element and the element consisting of content, padding and borders.
  • DRY - Do Not Repeat Yourself principle - combine or share pieces of code to reduce the number of lines written or maintained. Example: header, footer { - combined two selectors into one CSS rule with a comma.
  • CSS styles make a website visually appealing and simpler to use.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git commit -m: leaves a message regarding changes/debugging
  • git config --global user.email "you@example.com": set the account's default identity
  • git branch: lists user's branches
  • git log: shows all commits in the current branch history
