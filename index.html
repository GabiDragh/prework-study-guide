HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • !DOCTYPE declaration states the type of document the browser is expected to open
  • HTML attributes (lang="en") need to be defined at the beggining, not at the end of the element
  • Metadata tags include title, style, base, link, meta and script
  • Child elements are nested inside parent elements

CSS

  • Inline CSS - syle an element directly in the HTML file. Applies to specific HTML tag. Good for quick changes, but less flexible because each element needs editing individually.
  • Most frequent - external as it allows all design changes to be kept separate. It is brought in as a link element.
  • Separation of concerns principle - each section of code had its own responsibility
  • A CSS rule is formed of a selector and a declaration block. The selector defined the element or attributes to which the rules (declarations) will apply. Declarations are formed of a property and its value. Example Color is a property and blue is the value. Margin and padding are different properties. The margin states the amount of space around the outside of an element while padding refers to the space inside and element. The element gets bigger the more padding is added. CSS box model describes HTML elements as boxes with margins outside the element and the element consisting of content, padding and borders.
  • DRY - Do Not Repeat Yourself principle - combine or share pieces of code to reduce the number of lines written or maintained. Example: header, footer { - combined two selectors into one CSS rule with a comma.
  • CSS styles make a website visually appealing and simpler to use.

Git

  • git status: checks what branch we are currently on
  • git checkout main switch to the main branch
  • git pull origin main: Pull the remote main branch into the local main branch
  • git checkout -b branch-name: creates a new branch and switches to it
  • git commit -m: leaves a message regarding changes/debugging
  • git config --global user.email "you@example.com": set the account's default identity
  • git branch: lists user's branches
  • git log: shows all commits in the current branch history

JavaScript

  • A variable is a container where code data is stored.
  • VAR in JS decribes a variable. VAR name = "data"
  • var topics="HTML, CSS, Git, JavaScript", where "" includes a string in the variable topics.
  • All JavaScript lines end in semicolon ;
  • console.log(topics); - console.log is the method used to output a message to the web console by adding an argument (the value of topics).
  • Console = test environment for checking code.
  • .log = method where the computer executes a set of instructions and it outputs anything in the paranthesis to the console.
  • script.js file is linked in the element, unlike style.css in the .
  • Console.log method makes it easier for developers to test code and it is used daily to debug
  • Control flow - the order a computer executes codes in a script. In JS it executes top to bottom.
  • First we define topics then tell the computer to log it in the console. If not, 'undefined' appears in the console.
  • Control flow can be interrupted by conditional statements and loops that control the order in which the code is executed or if executed at all.
  • IF statement introduces decision-making into a code. if (condition) { // block of code to be executed if condition is truthy }
  • Falsy value - 0, -0, On, "", null, undefined, NaN or Boolean data type with the value of False. All others are truthy
  • === - stricy equality operator that checks if two values are equal (case sensitive).
  • Array - group of related data or a single variable used to hold a group of related data. Example: var studentInfo = ["Lu", 54, true]; - "Lu" is a string, therefore it is written between quotation marks.
  • Individual data items inside an array contain one more piece of information: an index (the location of the data item in the array). Index starts at 0.
  • FOR loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. It can be interrupted by an IF statement.
  • FOR loop - to use it we need the shape's indes and the array's lenght (inportant as the loops needs to know how many times the code will run until there are no more ites left in the array).
  • Lenght of an array - built-in property
  • FOR loop contains 3 statements: - starting point, condition and final statement. Statement separated by semicolon ; Example: var topics = ["HTML", "CSS", "Git", "JavaScript"]; for (var x = 0; x < topics.length; x++){ console.log(topics[x]); }
  • Commenting out code makes it inactive without deleting it. It needs /* */ before and after the block of code.
  • Function - set of instructions that tells the computer how to perform a certain task.
  • A function's name is followed by parantheses ().
  • The code to be executed is written between curly brakets {}. Example: function helloWorld() { console.log("Hello, world, I am a function!") }
  • Declaring a function is not enough for the code to execute it, instead we need to also call it in the place we need it to execute.
  • Math.random - helps select a random number Example: var randomTopic = topics[Math.floor(Math.random() * topics.length)]; var topics = ['HTML', 'CSS', 'Git', 'JavaScript'];
